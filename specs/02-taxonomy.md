# Spec: crate `taxonomy`

## Source Reference
Current Rust implementation: `crates/taxonomy/`
Historical rewrite input: `packages/taxonomy/` (tags.py, normalize.py)

## Purpose
Self-contained tag taxonomy crate for normalizing, validating, and suggesting Anki tags. Contains the canonical tag mapping (500+ entries mapping variant spellings to normalized topic tags), tag prefix definitions, and topic tag sets for Kotlin, Android, CompSci, and Cognitive Bias domains. Has zero external crate dependencies beyond the standard library -- all logic is pure functions over static data.

### Normalization (`src/normalize.rs`)

```rust
/// Normalize a single tag to canonical form.
///
/// Rules applied in order:
/// 1. Trim whitespace. Return `""` if empty.
/// 2. If tag is a standalone meta tag or starts with a meta prefix, return as-is.
/// 3. If tag starts with a known topic prefix (`kotlin_`, `android_`, `cs_`, `bias_`)
///    or equals `"cognitive_bias"`, return as-is.
/// 4. Look up in TAG_MAPPING. If found, return the mapped value.
/// 5. Otherwise: lowercase, replace `_`/`::`/`/` with `-`, collapse `--`, strip leading/trailing `-`.
pub fn normalize_tag(tag: &str) -> String;

/// Normalize a list of tags: normalize each, deduplicate, sort, remove empties.
pub fn normalize_tags(tags: &[&str]) -> Vec<String>;

/// Validate a tag and return a list of issues (empty if valid).
///
/// Checks:
/// - Not empty or whitespace-only.
/// - Uses `::` for domain prefix (not `_` or `/`).
/// - Kebab-case within parts (no underscores as word separators).
/// - Max 2 hierarchy levels (`prefix::topic`).
/// - Lowercase prefix.
/// - No duplicate separators (`::::`, `--`).
pub fn validate_tag(tag: &str) -> Vec<String>;

/// Suggest up to `max_results` close matches from the known taxonomy.
///
/// Algorithm:
/// 1. Collect candidates from TAG_MAPPING keys + values + all topic tags.
/// 2. Score each candidate:
///    - Skip exact matches.
///    - Prefix match (either direction) -> score 0 (best).
///    - If length difference > `max_distance`, skip.
///    - Character-difference count + length difference -> score.
///    - Keep if score <= `max_distance`.
/// 3. Sort by (score, candidate) ascending, take first `max_results`.
pub fn suggest_tag(input_tag: &str, max_results: usize) -> Vec<String>;

/// Check if a tag is a meta tag (starts with a meta prefix or is in META_TAGS).
pub fn is_meta_tag(tag: &str) -> bool;

/// Check if a tag is a recognized topic tag.
pub fn is_topic_tag(tag: &str) -> bool;
```

## Internal Details

### Tag mapping data
The `TAG_MAPPING` contains 500+ entries. The exact entries are defined in the Python source `packages/taxonomy/tags.py` in the `TAG_MAPPING` dict. Every key-value pair must be ported verbatim. Use `phf::phf_map!` or a build script to generate the lookup at compile time.

### Normalization edge cases
- `"cognitive_bias"` is special-cased: it starts with no standard topic prefix but is treated as a topic tag.
- Tags already in `ALL_TOPIC_TAGS` (e.g. `"kotlin_coroutines"`) pass through `validate_tag` without the "use :: not _" warning, because they are canonical underscore-format tags.
- The `suggest_tag` function uses a simple character-difference heuristic, not Levenshtein distance. It counts `sum(a != b for a, b in zip(lower1, lower2)) + abs(len1 - len2)`.

### No-alloc where possible
`is_meta_tag`, `is_topic_tag`, and `lookup_tag` should not allocate. `normalize_tag` returns a `String` because it may transform the input.
